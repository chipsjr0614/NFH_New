# statement-lint

간호기록의 모든 진술문은 **진술문 마스터(`Nr.statement_*.xlsx`의 `진술문` 시트)에 있는 문장만** 써야 한다.
이 스킬은 산출물에서 진술문을 추출해 마스터와 대조하고, 벗어난 항목을 잡아낸다.

## 언제 쓰나
- 사정 앱(`mockups/*.html`)이나 간호기록 데이터(`*_수정본.json/.xlsx`)를 새로 만들거나 고친 뒤
- 진술문 마스터가 갱신된 뒤 기존 산출물이 여전히 유효한지 확인할 때
- "이거 진술문 마스터 안 벗어났나?" 류의 요청

## 실행
```bash
python3 .claude/skills/statement-lint/lint.py <대상 ...> [--master PATH] [--sheet 진술문] [--strict]
```
- **대상**: `.json`(수정본/blocks 구조), `.xlsx`(수정본 포맷, `진술문` 컬럼 자동 인식), `.html`(앱에 임베드된 `BLOCKS`), `.txt`(줄당 진술문 하나). 여러 개 동시 가능.
- **--master**: 생략하면 현재 저장소에서 `Nr.statement*.xlsx`(또는 `*진술문*마스터*.xlsx`)를 자동 탐색하고 `0_참고문서/`·`refs/`를 우선한다. 파일이 이동돼도 알아서 찾는다.
- **--sheet**: 기본 `진술문`.
- **--strict**: 표기 변형(띄어쓰기·문장부호 차이)도 실패(exit 1)로 처리.

## 결과 읽는 법
- ❌ **마스터에 없음** — 진짜 벗어난 진술문. 가까운 후보를 제시하니 그걸로 고친다. (exit code 1)
- ⚠️ **표기 변형** — 띄어쓰기·문장부호·영문 대소문자만 다름(예: `사지근력 평가함` ↔ 마스터 `사지 근력 평가함`). 의미는 같지만 마스터 표기에 맞추는 게 원칙.
- ℹ️ **값 입력형** — `동공크기 :`처럼 `:`로 끝나는 플레이스홀더. 검증 제외.
- "있음/없음" 토글(`X 있음/없음`)은 자동으로 각 옵션(`X 있음`, `X 없음`)으로 펼쳐 검사한다.

## 종료코드
`0` 통과 · `1` 벗어난 진술문 있음 · `2` 마스터 못 찾음. CI나 커밋 전 게이트로 쓸 수 있다.

## 고칠 때 원칙
추출된 진술문을 마스터 문장으로 **그대로** 교체한다. 새 표현을 만들지 말 것 — 마스터에 없으면 마스터에 항목을 추가하는 게 맞는지 사람에게 확인한다.
