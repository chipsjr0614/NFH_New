# CLAUDE.md — AI 작업 규칙

이 저장소에서 작업하는 AI(Claude)가 지킬 규칙. **이름·폴더·버전·문서 규칙의 정본은 [`CONVENTION.md`](CONVENTION.md)** 이며 그것을 따른다. 프로젝트 맥락은 [`README.md`](README.md).
이 문서는 CONVENTION.md와 중복하지 않고, **AI가 추가로 지킬 작업 원칙**만 적는다.

---

## 0. 대원칙

- **응답은 한국어.**
- **★ 한글 파일 편집 시 `perl`/`sed` 등 명령줄 치환 금지.** 터미널 인코딩(NFD)이 한글을 모지바케로 깨뜨릴 수 있다(실제 사고 발생). 반드시 **Edit 도구** 또는 **Python(파일 읽기→치환→쓰기)** 만 사용. Python에서도 한글은 가급적 코드포인트보다 파일 내용 기반으로 다룬다.
- **문서가 먼저다.** 오너는 문서를 매우 중요하게 여긴다. 코드를 바꾸면 관련 `<주제>_요구사항정의서.md`·`README.md`도 같은 작업에서 갱신한다. 문서 없는 기능 추가 금지.
- **의료 안전 우선.** 임상 사실(용량·금기·검사 기준 등)을 **추측해서 지어내지 않는다.** 불확실하면 명시하고 **칩스(수간호사) 검증 필요**로 표시한다.

> ## ⛔ 절대 규칙 — 데이터 무결성 (타협 불가)
>
> **데이터를 조작하거나 지어내지 말라는 지침이 있는 경우, 그 지침을 절대적으로 지킨다. 예외 없음.**
>
> - 원본 데이터(엑셀·진술문·임상값 등)를 **임의로 수정·보정·생성하지 않는다.** 받은 그대로 다룬다.
> - 값이 **없거나 불확실하면 비워두거나 "확인 필요"로 표시.** 그럴듯한 값으로 채우지 않는다.
> - 샘플·플레이스홀더가 불가피하면 **반드시 가짜임을 명시**하고 실제 데이터와 섞지 않는다.
> - 데이터가 부족해 막히면 추측으로 메우지 말고 **멈춰서 사용자에게 알린다.**
> - 이 규칙은 편의·진행 속도보다 우선한다. 의심스러우면 지어내지 않는 쪽으로.

## 1. 이름·폴더·버전·문서 → `CONVENTION.md` 따름

상세는 CONVENTION.md. AI가 특히 기억할 요지만:
- 새 파일은 **해당 서비스 폴더**(`3_서비스/<서비스명>/`) 안에. 배포본은 항상 `app/index.html`.
- 문서는 한글 `<주제>_<문서종류>` (제목만 봐도 알게). **★ 내가 생성하는 문서는 파일명 끝에 `_YYMMDD`** (예: `간호기록_사정흐름_260613.md`).
- `1_회의노트/`는 **불변 원본** — 읽기만, 수정·삭제 금지.
- `README.md`·`CLAUDE.md`·`.gitignore`는 도구 강제 파일이라 영문·무날짜 유지(`_YYMMDD` 제외).

## 2. 기술 제약 (병원 환경)

- **인터넷 차단(내부망).** 런타임에 외부 네트워크·AI API를 호출하는 코드를 넣지 않는다. 데이터·질문 세트는 **빌드 시점에 앱에 인라인.**
- 앱은 **자체완결형 단일 HTML** 지향(현재 모든 앱이 데이터 인라인). 외부 fetch 의존 추가 지양.
- 대상 기기: **태블릿 기준 + 핸드폰/PC 반응형.**
- `_archive/`의 `build_*.py`는 과거 머신 절대경로(`/Users/sungkwanchoi/...`)에 묶여 **실행 불가** — 참고용, 되살려 쓰지 않는다.

## 3. 데이터 규칙

- 간호기록의 핵심은 **"7만 항목" 정형 간호 진술문**(예: "토했음"❌ → "오심 구토 있음"✅). 임의 자연어로 바꾸지 말고 **정형 문구를 따른다.**
- 마스터 데이터 일부는 아직 저장소에 없을 수 있다(칩스 보유). 없으면 지어내지 말고 누락을 명시. (→ 0번 절대 규칙)

## 4. Git 규칙

- **커밋·푸시는 사용자가 요청할 때만.** 묻지 않고 자동 커밋하지 않는다.
- 커밋 메시지·문서·코드 주석은 **한국어.**
- 큰 구조 변경은 `git mv`로 **히스토리 보존.**
