# 간호기록 알고리즘 시스템 — 요구사항 정의서

**버전:** v3.0  
**작성일:** 2026-06-06  
**대상:** 국립소방병원 응급실 신입·전입 간호사  
**파일:** `nursing_note.html` (단일 HTML, 오프라인)  
**데이터:** `nursing_data.json` — 원본 그대로만 사용 (임의 변형 금지)

---

## 1. 핵심 원칙

| 원칙 | 내용 |
|------|------|
| 데이터 출처 | nursing_data.json 원문만 사용 |
| 환자 설명·교육 스크립트 | 앱 내 별도 정의 (간호기록에는 미포함, 안내용) |
| 간호진단 | 화면 미표시 — 팝업에만 자동 추가 |
| 의식/낙상 | 항상 마지막, 제외 불가 🔒 |
| 최종 출력 | 팝업 체크리스트 (보면서 EMR 직접 입력) |

---

## 2. 전체 흐름 (5단계)

```
[STEP 1] 침습적 도관 확인
         ↓
[STEP 2] 주증상 선택 (복수 선택 가능)
         ↓
[STEP 3] 분기 질문 (증상별 소분류)
         ↓
[STEP 4] 사정·교육·계획 (한 화면, 증상당 반복)
         ── 사정 【A&E】 간호진술문
         ── 📢 환자 설명 (낙상교육 스타일)
         ── 계획 및 평가 【P&E】 간호진술문
         ↓
[STEP 5] 의식/낙상 교육 (항상 포함)
         ↓
[완료]   간호기록 - 간호일지 팝업
```

---

## 3. STEP 1 — 침습적 도관

풀텀 + 약어 표시. 없으면 건너뜀.

| 표시명 | 기록 진술문 |
|--------|------------|
| Central Venous Catheter (CVC/PICC) | Central Venous Catheter (CVC/PICC) 삽입중 |
| Foley Catheter (유치도뇨관) | Foley Catheter (유치도뇨관) 삽입중 |
| Percutaneous Transhepatic Biliary Drainage (PTBD) | PTBD 삽입중 |
| Percutaneous Transhepatic Gallbladder Drainage (PTGBD) | PTGBD 삽입중 |
| Percutaneous Catheter Drainage (PCD) | PCD 삽입중 |
| 흉관 유지중 | 흉관 유지중 |
| 기타 직접 입력 | 입력값 그대로 |

---

## 4. STEP 2 — 주증상 (15개)

😣통증 / 🫁호흡곤란 / 🩸출혈(눈에 보임) / 🩹출혈(외상·코피) / ⚠️출혈위험  
🦠감염 / 🚨감염위험 / 🌡️고체온 / 🥶저체온 / 🧠뇌경색·뇌출혈  
❤️심근경색 의심 / 🍬고혈당 / 💉저혈당 / ⚡Seizure / ✅이상없음

- 복수 선택 가능 → 각 증상별로 STEP 4 반복
- 이상없음 선택 → STEP 5로 바로 이동

---

## 5. STEP 3 — 분기 질문

| 주증상 | 분기 질문 |
|--------|-----------|
| 통증 | 부위? (복부/흉부/두통/기타) |
| 호흡곤란 | 가래 많음? 심부전 의심? 흉통 동반? |
| 출혈(눈에 보임) | 종류? → 각 통증 동반 여부? |
| 출혈(외상) | 통증 동반? |
| 출혈위험 | 통증 동반? |
| 고체온 | 통증 동반? |
| 심근경색 | 흉통 동반? |
| Seizure | 지금 발작 중? / 멈추고 내원? |
| 그 외 | 단일 세트 |

## 6. STEP 4 — 사정·교육·계획 (핵심)

### 6-1. 화면 구조 (한 화면에 세 섹션)

```
사정 【A&E】 간호진술문      ← 파란색 헤더
  💬 질문 스크립트 + 탭 버튼
  (슬래시 → 두 버튼 / 콜론 → 텍스트 입력 / 일반 → 체크박스)

📢 환자 설명                 ← 낙상교육 스타일
  말할 문장 표시 → 체크 시 P&E 기록 연동

계획 및 평가 【P&E】 간호진술문  ← 노란색 헤더
  체크박스 (nursing_data.json 원문)
```

### 6-2. 통증 전용 — NRS 평가 도구

통증 있음 선택 시 자동 펼침:

| 평가 항목 | 입력 방식 | 기록 진술문 예시 |
|-----------|-----------|-----------------|
| 통증강도 | NRS 0~10 탭 버튼 | 통증강도 7점 |
| 통증부위 | [상복부][하복부][우하복부][좌하복부][전체] | 통증부위 하복부 |
| 통증양상 | [쥐어짜듯][찌르듯][둔하게][타는듯][콕콕] | 통증양상 쥐어짜듯 |
| 발생시기 | [방금 전][몇 시간 전][어제][며칠 전] | 통증기간 몇 시간 전부터 |
| 지속여부 | [지속적으로][간헐적으로][지금은 없음] | 통증지속 지속적으로 |

> 모든 통증 증상(복부/흉부/두통/기타/출혈동반통증 등)에 동일 적용

### 6-3. 📢 환자 설명 스크립트 (증상별)

화면에 말할 문장 표시 → 체크 시 해당 P&E 항목이 기록에 포함

**통증:**
- "많이 아프세요? 진통제가 필요하실 만큼 아프신가요?" → 통증을 인정해줌
- "의사 선생님께 확인받고 필요하시면 진통제 처방받아 드릴게요." → 통증 양상을 확인함
- "참으실 수 있으면 조금 지켜보시고, 너무 아프시면 바로 말씀해 주세요." → 통증을 관찰하기로 함

**호흡곤란:**
- "숨이 많이 차세요? 지금 산소 수치 확인해 드릴게요." → 호흡곤란이 있는지 확인함
- "산소 마스크 착용해 드릴게요. 더 힘드시면 바로 말씀해 주세요." → 산소흡입 시작함

**출혈:**
- "지금 피가 나오는 상태를 확인할게요. 증상이 심해지면 바로 알려주세요." → 출혈 증상이 있는지 확인함

**신경계 (뇌경색/뇌출혈):**
- "지금 팔다리 힘이 괜찮으세요? 감각이 이상한 부분은 없으세요?" → 신경학적 증상이 있는지 확인함
- "최대한 안정하시고 증상 변화가 있으면 바로 말씀해 주세요." → 사지근력 평가함

### 6-4. 간호진단 처리

- 화면에 표시 안 함
- nursing_data.json D 항목 → 자동으로 팝업에만 추가
- 낙상위험은 항상 포함 🔒

## 7. STEP 5 — 의식/낙상

### A. 의식 확인
💬 "성함이 어떻게 되세요? 오늘 날짜 아세요? 여기가 어딘지 아세요?"

| 선택 | 기록 |
|------|------|
| 의식 명료함 | 의식 명료함 |
| 의식 저하 | 의식수준 : [입력] / Obey command / 지남력 |

### B. 보호자 확인
| 선택 | 기록 |
|------|------|
| 보호자 상주 | 보호자 환자 옆에 상주하고 있음 |
| 보호자 없음 | 보호자 동행하지 않음 |

### C. 낙상교육 스크립트
말할 문장 표시 → 체크 시 nursing_data.json P&E 원문 기록

| 📢 말할 문장 | 기록 (원문) |
|-------------|------------|
| "침대에서 내려오실 때는 반드시 간호사를 불러주세요." | 침대에서 내려오실 때 반드시 간호사 호출하도록 교육함 |
| "어지러우시면 바로 앉으시고, 보호자나 담당 간호사에게 알려주세요." | 어지러우면 바로 앉고 보호자나 담당간호사에게 알리도록 교육함 |
| "낙상 위험이 있어서 침대 난간 올려드릴게요." | 낙상 위험성에 대해 설명함 / 침대 난간을 올려줌 / 낙상방지 교육함 |
| (보호자 있을 때) "옆에서 계속 함께 있어 주세요." | 보호자와 같이 있게 함 |
| (보호자 없을 때) "보호자분 연락해 주세요." | 보호자 동반하도록 설명함 |

## 8. 팝업 — 간호기록 - 간호일지

```
간호진술문                      ← 도관 있을 때만
  · [도관 기록]

[증상명] — 간호진술문
  사정 【A&E】 간호진술문        ← 파란색
  · [A&E 체크 항목들]
  · [NRS 평가 결과 줄줄이]

  진단 【D】                    ← 빨간색 (자동)
  · [진단명]

  계획 및 평가 【P&E】 간호진술문 ← 노란색
  · [체크한 P&E 항목들]

의식/낙상 — 간호진술문
  사정 【A&E】 간호진술문
  · 의식 명료함 / 보호자 상주 등

  진단 【D】
  · 낙상위험 (🔒 항상)

  계획 및 평가 【P&E】 간호진술문
  · [낙상교육 체크 항목들]
```
