# 컨벤션 (정본)

폴더·이름·버전·문서 규칙의 단일 진실 원천(정본). 사람과 AI 모두 이 문서를 따른다.
AI가 추가로 지킬 작업 원칙은 [`CLAUDE.md`](CLAUDE.md).

## 1. 이름 규칙 — 영문/한글 혼용

| 대상 | 규칙 | 예 |
|------|------|-----|
| 최상위 폴더 | **`번호_한글명`** (번호로 정렬·역할 구분) | `0_참고문서`, `1_회의록`, `2_기획문서`, `3_서비스`, `9_기타문서` |
| 도메인 서비스명 폴더 | **한글** | `간호기록`, `약물계산기` (※ `ER Mate`·`triage`는 통일 검토 필요) |
| 하위 구조 폴더 | **영문 소문자** | `app`, `mockups`, `refs`, `assets`, `_archive` |
| 코드·앱 파일 | **영문 소문자** | `index.html`, `er_mate_data.json` |
| 문서 파일 | **한글**, `<주제>_<문서종류>` 형식. 공백 대신 `_` | `간호기록_요구사항정의서.md`, `약물_참고문헌.md` |
| 스냅샷 문서 | `<주제>_<문서종류>_YYYY-MM-DD` | `현황정리_2026-06-13.md` |
| 참고 원본 자료 | 받은 이름 유지 가능 | `병용투여 가능 항생제.pptx` |

**번호 체계:** `0`=원천 데이터·참고자료, `1`=회의록(불변 원본), `2`=기획문서, `3`=서비스(제품), `9`=기타·백로그.
**금지:** 하위 구조 폴더와 코드 식별자에 공백·대문자·한글. (URL·정렬·도구 호환 문제)
**예외:** 최상위 폴더와 도메인 **서비스명 폴더는 한글** 허용(`3_서비스/간호기록`). 단 그 안의 `app`·`refs`·`_archive`와 코드 파일은 영문.

### 핵심 원칙 — 제목만 봐도 무슨 파일인지 안다

문서명은 **폴더 밖에서 봐도** 내용을 추측할 수 있어야 한다. 그래서 문서종류(요구사항정의서·참고문헌 등)
앞에 **반드시 주제 접두사**를 붙인다. `요구사항정의서.md`(❌ 폴더 안에서만 통함) → `간호기록_요구사항정의서.md`(✅).

| 형식 | 용도 | 예 |
|------|------|-----|
| `<주제>_요구사항정의서.md` | 서비스/앱 요구사항(SSOT) | `트리야지_요구사항정의서.md`, `약물계산기_요구사항정의서.md` |
| `<주제>_참고문헌.md` | 임상·기술 근거 자료 | `약물_참고문헌.md` |
| `현황정리_YYYY-MM-DD.md` | 특정 시점 현황 스냅샷 (날짜 필수) | `현황정리_2026-06-13.md` |
| `컨벤션.md` / `로드맵.md` | 프로젝트 단일 전역 문서(주제가 곧 제목, 접두사 불필요) | — |

- **스냅샷·회의록처럼 "그 시점"이 의미 있는 문서는 파일명에 날짜를 넣는다.**
- 계속 갱신되는 산출물(요구사항정의서 등)은 파일명에 날짜를 넣지 않고, 문서 안에 `버전`/`작성일`을 적는다. 변경 이력은 git이 관리.
- **★ AI(Claude)가 새로 생성하는 문서는 파일명 끝에 항상 `_YYMMDD`(2자리 연도)를 붙인다.** 형식 `<주제>_<문서종류>_YYMMDD.md`, 예: 2026-06-13 생성 → `간호기록_사정흐름_260613.md`. (도구 강제 파일 `README.md`·`CLAUDE.md` 등은 제외)

**예외 — 도구가 이름을 강제하는 영문 파일 (바꾸면 동작 깨짐, 그대로 유지):**
- `README.md` — GitHub가 폴더 진입 시 자동 렌더링하는 표준 개요 파일
- `CLAUDE.md` — Claude Code가 그 이름으로 자동 로드하는 규칙 파일
- `.gitignore` 등 점(.)으로 시작하는 도구 설정 파일

## 2. 폴더 구조

```
chips/
├─ README.md / CLAUDE.md
├─ 0_참고문서/   원천 데이터·공용 참고자료 (NFH 마스터 엑셀 등)
├─ 1_회의록/     회의 녹음·사진 (불변 원본, 수정 금지)
├─ 2_기획문서/   전역 문서 (현황정리·컨벤션·로드맵, _archive)
├─ 3_서비스/     서비스 4개 (간호기록·약물계산기·ER Mate·triage)
└─ 9_기타문서/   향후 후보·기타 (인계장공유, 간호교육자료)
```

### 서비스 폴더 공통 레이아웃
```
3_서비스/<서비스명>/
├─ README.md               한 줄 정의 + 현재 상태 + 다음 작업 (필수)
├─ <주제>_요구사항정의서.md  해당 서비스 SSOT (필수) 예: 간호기록_요구사항정의서.md
├─ app/                    배포 산출물: index.html (항상 최신)
├─ mockups/                디자인 후보·UI 시안 (검토 중, 선택)
├─ refs/                   참고 자료 pptx/docx/설계서 (선택)
└─ _archive/               구버전·폐기 시도 보존 (선택)
```

### 0_참고문서 vs refs/ 구분
- `0_참고문서/` : 앱에 들어갈 **구조화된 마스터 원천**(간호기록 7만 항목, 약물간호 엑셀 등) + 공용 참고자료.
- `3_서비스/*/refs/` : 그 서비스 전용 **참고·근거 자료**(투약방법 pptx, DB 설계서 등).

## 3. 버전 관리 — Git + 고정 파일명

핵심 규칙: **파일명에 버전을 박지 않는다. 이력은 git이 책임진다.**

| 상황 | 방식 |
|------|------|
| 현재 작업본 | 항상 고정 이름 (`app/index.html`) |
| 이전 버전 보고 싶을 때 | `git log` / `git diff` |
| 폐기했지만 보존할 시도 | `_archive/`로 이동 (삭제 ❌). 아카이브 내부에서만 `_v1` 등 허용 |
| 외부에서 받은 원본 엑셀 | 받은 파일명(날짜 포함) 그대로 — 손대지 않음 |
| 스냅샷 문서(현황정리·회의록) | 파일명에 날짜 명시 (`_YYYY-MM-DD`) |

**구분:** 날짜는 **"그 시점"이 의미인 스냅샷 문서**에만 붙인다. 코드/앱(`index.html`)이나 계속 갱신되는 산출물에는 버전·날짜를 박지 않는다(= 이력은 git).
**안티패턴(하지 말 것):** `er_mate_v2.html`, `간호기록_수정본.html`, `index_최종_진짜최종.html`

## 4. 문서 규칙

- 모든 서비스는 `README.md` + `<주제>_요구사항정의서.md`를 가진다.
- `<주제>_요구사항정의서.md`는 그 서비스의 **단일 진실 원천(SSOT)**. 기능 변경 시 코드와 함께 갱신.
- 전역·교차 문서는 `2_기획문서/`. 회의 스냅샷은 `현황정리_YYYY-MM-DD.md`로 날짜를 남긴다.
- 문서·주석·커밋 메시지는 한국어.
