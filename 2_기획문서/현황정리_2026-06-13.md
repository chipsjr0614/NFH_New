## 0. 한눈에 보기

응급실 간호 업무를 돕는 앱을 **3개**로 쪼개서 만들고 있음. 셋 다 같은 뿌리(=ER메이트에 다 넣고 싶었던 것)에서 갈라져 나왔고, 환경은 **태블릿(아이패드) 기준 + 핸드폰/PC 반응형**, 병원 내부망은 **인터넷 차단(방화벽)** 이라 AI 호출은 못 하고 데이터를 **사전에 다 넣어둬야** 함.

| # | 서비스 | 한 줄 정의 | 현재 상태 | 이번 우선순위 |
|---|--------|-----------|----------|--------------|
| 1 | **트리야지 (초진/KTAS)** | 응급실 입구에서 5분 내 초진·중증도 분류 | **완성, 월요일부터 실사용 중** | 낮음 (팔찌/이름 팝업만 추가) |
| 2 | **간호기록 (NFH)** | 담당 간호사가 환자 사정 → 간호기록 자동 생성 | **버그 많음, 처음부터 재설계 필요** | **최우선 (가장 중요)** |
| 3 | **ER메이트** | 응급실 전반 체크리스트 + 약물 계산기 | UI 가장 발전, 데이터 업데이트 필요 | 중간 |

## 1. 트리야지 (초진 / KTAS 툴)

**무엇:** 환자가 응급실 입구에 도착했을 때 담당 = "초진 간호사"가 **5분 이내(급하면 1~2분)** 로 빠르게 보는 중증도 분류 도구.

**흐름 (녹음 시나리오 기준):**
1. 접수 시간(AMR 등록 8자리) + 환자 입력
2. 첫인상(appearance) 체크 — 숨 잘 쉬나, 말하나, 걸어왔나, 얼굴/땀
   - 문제 있으면 **바로 급성(소생실)으로 집어넣음**, 괜찮으면 진행
3. 활력징후(혈압·맥박·의식) 확인 — 의식 없으면 **혈당 먼저** (저혈당 거르기)
4. 주호소 문진 — "왜 오셨어요?" + 위험징후(토혈/혈변/임신가능성/통증점수/발병시점 등)
5. 과거력·수술력·투약력 확인
6. **KTAS 레벨 판정** (예: 레벨 3 = 1시간 내 진료)
7. 결과를 **한 줄 텍스트로 복사** → "초진 기록일지"에 붙여넣기 → 의사·로그 공유

**핵심 의도:** 간호사마다 문진이 **중구난방**인 걸 표준화. 일을 두 번 하지 않게 한 번에.

**상태:** **완성. 월요일부터 현장에서 실사용 중. 큰 문제 없음.**

**남은 작업:**
- 시작 시 **환자 확인 팝업**(팔찌 + 이름 둘 다 확인) 추가 — "그냥 진짜 팝업만 넣으면 됨" 수준
- 디테일 문진 항목 약간 보강
- (화이트보드) 입력 방식 / TB·Phone 반응형 / "앱 끝내고 5분 안" 메모

## 2. 간호기록 (NFH) — ★최우선

**무엇:** 환자가 구역(관찰 구역 / 급성 구역)에 들어와 누우면 **담당 간호사**가 환자를 사정(assessment)하고, 그 결과로 **정형화된 간호기록을 자동 생성**하는 도구. 초진보다 깊게(10~20분) 봄.

**핵심 컨셉 — "스무고개" 방식 사정 → 기록 자동화:**
1. **환자 확인 팝업** (시작 전): "성함이? 팔찌 확인하겠습니다 → 확인됨. 저는 담당 간호사 ○○입니다. 금식하세요." (멘트 대본도 제공하고 싶음)
2. **외부 도관 확인**: 주사바늘/IV, 콧줄, 소변줄 등 (타 병원에서 달고 올 수 있으니 1차로 거름)
3. **주호소 → 증상 카테고리화 문진**: "뭐 때문에 오셨나요?" → 예: 복통 클릭 시
   - 구역/구토 있음? 설사? 열감? 혈변/흑색변? 임신가능성? 통증(부위·양상·점수·발병시점)? 등 **딱 필요한 10개 항목**만
   - **채팅형 UI**: 항목 입력 시 "없음"이면 그냥 넘어가고, "있음"이면 세부(횟수/음식물/투사성 등) 입력칸 활성화
   - 추가 증상(머리·가슴 등) 있으면 같이 처리, AI가 간호기록 분석해 추가 질문
4. **사정(Assessment)** → **간호진단(Nursing Dx)** (예: 급성 통증) → **평가 및 중재** (금식 안내, 검사/투약 가능성, 진통제 확인 등)
5. **낙상 예방 교육**(디폴트): 침대 올리기, 보호자 부재 시 주의, 화장실 동행 안내 — *빠뜨리면 처벌 대상이라 필수*
6. (예외) "약만 맞으러 온 사람" 등 **이상 없음** 선택 시 문진 건너뛰기

**데이터 핵심 — "7만 가지 간호기록":**
- 병원에서 실제 쓰는 **정형화된 간호 진술문**(예: "토했음"❌ → "오심 구토 있음"✅)이 약 7만 항목 존재(진단명 포함).
- 응급실/공통/수술실·중환자실 전용으로 나뉘므로 **응급실+공통만 카테고리화**하면 됨 → 실제 문진 항목은 **100~200개 이하**.
- AI가 이 데이터를 분석·카테고리화해서 질문을 만들고, 답변(클릭)을 바탕으로 **정형 간호기록을 출력**.
- ⚠️ 데이터는 현재 **집에 있고 레포에 없음**. 사전 주입 필요.

**왜 자동화:** ① 신규 간호사가 **뭘 물어봐야 할지 모름**, ② 정형 진술문을 **쓸 줄 모름**, ③ 한 항목 놓쳐서(혈당·임신 등) 불필요한 CT/MRI 찍는 사고 방지.

**상태:** 사정 툴 + 간호기록 툴을 **합치다가 망가짐**. 버그 多(뒤로가기 불가, 안 보임, 클릭 안 하면 문장 생성 안 됨). → **처음부터 다시 만들기로 결정.**

**진행 방식 (합의):**
- 구민정이 **기존 결과물을 문서(요구사항)로 다시 정리** → 처음부터 재구축. (현재 상태에 덧대면 계속 깨짐)
- 시스템 연동은 안 됨(간호기록 시스템과 분리) → 당장은 **옆에 띄워놓고 참고**. 먼 미래엔 저장 시 바로 간호기록으로 넘어가는 게 이상적.
- (선택) **복붙 공유**: 환자 이름/번호 + 기록을 한 덩어리로 복사 → 네이버 웍스로 공유. 필수는 아님(태블릿 로그인 이슈 고민 필요).

## 3. ER메이트

**무엇:** 응급실 전반의 체크리스트 + 도구 모음. 세 서비스의 **모태**(원래 다 여기 넣으려던 것). UI가 가장 발전돼 있음. 이미 외부에서 상업화한 사람(현직 간호사)이 있음.

**구성:**
- **약물 계산기** — **완성**. 몸무게 입력 시 용량 자동 계산(소아 특히 중요). "집어넣기만 하면 됨."
- **약물 정보** — 자주 쓰는 주요 약 검색/조회. (요구사항서엔 52개 기본 틀, 18개 카테고리)
- **항생제 병용 투여** — 항목 존재
- **검사/시술 준비 체크리스트** — 영상검사(CT·MRI 등). **데이터 업데이트 필요** + **각 항목에 "이유" 추가하고 싶음**
  - CT: 조영(contrast)/비조영 분기 → 조영이면 주사바늘(손목 위 20G↑, 터지면 괴사), 동의서, GFR 60↑(신장), 조영제 알러지, 전처치, 임신여부 / 비조영이면 임신여부만
  - MRI: 금속 제거가 핵심(보청기·목걸이·속옷·심전도 스티커 등 — 간호사가 직접 확인)
- **혈액검사 체크리스트** — 추가하고 싶음(시술검사와 별도, 종류 다양)
- **필수 업무 체크리스트** — 입원 / 퇴원 / **전원(타 병원으로 보냄)** / 사망
- **업무 가이드** — 자주 하지만 놓치는 업무. 예: **수혈**(ABO 오더 2개·꼬리표·IV 싱글/20G↑·알러지·시간제한 등)

**개선 방향:** 단순 체크박스(있냐/없냐)를 넘어서, **간호기록처럼 스무고개식으로 물어보며** 왜 필요한지(이유)까지 보여주는 업그레이드 버전.

**상태:** 세팅 잡혀 있고 데이터 일부 있음. **필요한 건 데이터 업데이트 + 체크리스트에 이유 추가** 정도.

## 4. 공통 환경 / 제약

- **기기:** 태블릿(아이패드) 기준 + 핸드폰/PC **반응형**. 현장에선 아이패드 들고 다니며 사용 중.
- **네트워크:** 병원 내부망 **인터넷 차단(방화벽)**. → 실시간 AI 호출 불가, **데이터·질문 세트를 사전 생성·주입**해야 함. AI는 개발 단계에서 카테고리화/질문 생성에 활용.
- **시스템 연동:** 기존 병원 간호기록 시스템과 **연동 불가(분리)**. 당장은 참고용으로 옆에 띄우거나 복붙 공유.
- **커뮤니케이션:** 현재 네이버 웍스 사용. 복붙 공유 아이디어의 연결 지점.
- **핵심 사용자:** **신규 간호사 / 타 병원 출신**. 경력직은 차트만 봐도 되지만 신규는 길라잡이 필요.
