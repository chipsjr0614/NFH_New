### 2단계 — 빠른 마감 (동작하는 것들)
- **트리야지**: 환자 확인 팝업(팔찌+이름) 추가 → 마감
- **약물 계산기**: 확인 절차 점검 후 ER메이트/단독 배포 경로 확정
- **ER메이트**: 마스터 데이터 업데이트 + 체크리스트에 "이유" 추가

### 3단계 — 간호기록 재구축 (핵심)
1. 기존 `간호기록노트.html`(`_archive/`)과 통합 시도(`ER_Mate_V3`)를 요구사항으로 역정리
2. "사정 → 진단 → 평가/중재 → 낙상교육" 흐름을 채팅형으로 재설계
3. 7만 정형 진술문을 카테고리화(응급실+공통만, 100~200 문항)해 질문 세트 사전 생성
4. 답변(클릭) → 정형 간호기록 자동 출력. 처음부터 새로 구현 (`app/index.html`)

### 4단계 — 운영·확장
- 4개 앱 공통 UI/디자인 통일, 반응형 점검(태블릿·폰·PC)
- 내부망 배포 패키지(오프라인 단일 HTML) 정리
- `backlog/` 승격 검토: 인계장 공유, 간호 교육자료
