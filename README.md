# CHIPS — 국립소방병원(NFH) 응급실 간호 지원 앱

응급실 간호 업무를 돕는 **4개 서비스**를 개발·운영하는 저장소입니다.
신규·타 병원 출신 간호사가 응급실 프로토콜을 놓치지 않도록 돕는 것이 전체 목표입니다.

> 도메인 기획·검증: **칩스**(응급실 수간호사) · 개발: **구민정**

---

## 서비스 한눈에 보기

| 서비스 | 폴더 | 한 줄 정의 | 상태 |
|--------|------|-----------|------|
| **트리야지** (초진/KTAS) | [`services/트리야지`](services/트리야지) | 입구에서 5분 내 초진·중증도 분류 | ✅ 실사용 중 |
| **간호기록** (NFH) ★ | [`services/간호기록`](services/간호기록) | 환자 사정 → 정형 간호기록 자동 생성 | 🔴 재설계 중 (최우선) |
| **ER메이트** | [`services/ER메이트`](services/ER메이트) | 응급실 전반 체크리스트 | 🟡 데이터 업데이트 필요 |
| **약물 계산기** | [`services/약물계산기`](services/약물계산기) | 체중 기반 약물 용량 계산(소아 PALS 포함) | 🟢 계산기 완성 |

각 서비스 폴더의 `README.md`에 상세 상태·다음 작업이 있습니다.

---

## 저장소 구조

```
chips/
├─ README.md              ← (이 파일) 전체 진입점
├─ CLAUDE.md              ← AI 에이전트가 지켜야 할 작업 규칙
├─ docs/                  ← 전역 문서
│   ├─ 현황정리_2026-06-13.md   회의 기반 현황 스냅샷
│   ├─ 컨벤션.md             폴더·이름·버전 규칙 (정본)
│   ├─ 로드맵.md             장기 진행 계획
│   └─ _archive/            (구) 통합본 요구사항 등 폐기 문서 보존
├─ services/             ← 서비스 4개 (각자 독립, 서비스명은 한글)
│   ├─ 트리야지/
│   ├─ 간호기록/
│   ├─ ER메이트/
│   └─ 약물계산기/
├─ data/                 ← NFH 마스터 엑셀 원본 (앱에 들어갈 원천 데이터)
├─ raws/                 ← 회의 녹음·화이트보드 사진 (불변 원본, 수정 금지)
└─ backlog/              ← 향후 후보 (인계장 공유, 간호 교육자료)
```

### 서비스 폴더 공통 레이아웃
```
services/<서비스>/
├─ README.md          서비스 설명 + 현재 상태 + 다음 작업
├─ <주제>_요구사항정의서.md  해당 서비스의 단일 진실 원천(SSOT)
├─ app/               배포 산출물 (index.html 등, 항상 최신)
├─ refs/              참고 자료 (pptx·docx 등, 있을 때만)
└─ _archive/          구버전·폐기된 시도 (참고용)
```

## 시작하기

- 각 앱은 **자체완결형 HTML** (데이터 인라인). 브라우저에서 `app/index.html`을 바로 열면 됩니다.
- 병원 환경은 **인터넷 차단(내부망)** — 런타임에 외부 호출/AI 불가. 모든 데이터·질문 세트는 사전 주입.
- 기기: **태블릿(아이패드) 기준 + 핸드폰/PC 반응형.**

작업 규칙은 [`CLAUDE.md`](CLAUDE.md)와 [`CONVENTION.md`](CONVENTION.md)를 먼저 읽어주세요.
